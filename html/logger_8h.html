enum LOG_LEVELS

The available logging Levels.

This will control what is written to logging files ; those files are named :

  • error.log
  • warning.log
  • info.log
  • debug.log

The higher the level, the more text is written to those text files, so be careful with long simulations coupled to high levels of logging.

The different levels, members of the enum LOG_LEVELS are :

  • LOG_NOTHING : At this level there is no logging at all.
  • LOG_ERROR : An Error is a critical event which will cause the program to abort immediately, for example a missing parameter for which no default value is provided.
  • LOG_WARNING : A Warning is a non-critical event, i.e. the event is just reported but the program execution continues, for example a missing or wrong parameter for which a default value is going to be used.
  • LOG_INFO : An Info message reports to the user a possibly useful information, an event with no consequence for the simulation. For example, that the trajectory was succesfully written as planed.
  • LOG_DEBUG : A debug message is really technical and verbose, for example dumping a variable. This should be enabled for bug tracking.

The logging is progressive, enabling LOG_INFO means that higher levels LOG_ERROR and LOG_WARNING are also enabled.

The default is set to LOG_WARNING.

Enumerator
LOG_NOTHING 

No log file is created : this is not recommended as no information is reported

LOG_ERROR 

Only errors are reported to error.log

LOG_WARNING 

Warnings are reported to warning.log , and also errors to error.log. This is the default.

LOG_INFO 

Info messages are reported to info.log , and also warnings and errors to their respective files.

LOG_DEBUG 

A large amount of debugging messages are written to debug.log . The previous levels are still written to their respective files.
